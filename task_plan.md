## Key Questions
1. 统计页展示哪些指标最适合答辩讲解且实现成本低？
2. 如何让页面展示既贴近“通用提示注入防护框架”，又不过度陷入具体业务细节？

## Decisions Made
| Decision | Rationale |
|----------|-----------|
| 采用服务端渲染而不是前后端分离 | 两周周期内交付更稳，更适合毕设演示 |
| 用户界面优先做聊天页、审计页、规则统计页 | 这三块最能完整展示“检测-决策-防护-留痕-分析”闭环 |
| 当前规则通过 Spring Bean 固定注入 | MVP 阶段优先保证可跑可演示，后续可再切到外部配置 |
| 每完成一小块功能就单独提交 | 方便保留开发记录，也方便用户回顾进度 |
| 在规则页同时展示规则表和关键指标卡片 | 对老师更直观，方便讲“系统如何识别”和“系统效果如何” |
